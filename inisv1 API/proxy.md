## 代理服务 API

### 接口概述

代理服务控制器，提供 HTTP 请求代理功能。

### 状态码规范

| 状态码 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 代理请求成功 |
| 202 | 接受请求 | 无实际功能接口 |
| 405 | 方法不允许 | 调用了不存在的方法 |
| 500 | 服务器错误 | 代理请求失败 |

### 接口列表

#### 1. 代理请求 [index]

**请求方式**: `GET`

**请求地址**: `/api/proxy`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| i-url | string | 是 | 目标URL |
| i-type | string | 否 | 返回类型(JSON/TEXT/BYTE)，默认JSON |

**响应示例**:

```json
{
  "code": 200,
  "msg": "好的！",
  "data": {
    "response": "目标服务器返回的数据"
  }
}
```

### 特殊说明

1. **代理功能**: 将请求转发到指定的 `i-url`，并返回目标服务器的响应
2. **返回类型**: 支持 JSON、TEXT、BYTE 三种格式
3. **请求透传**: 请求的 Headers、Query、Body 参数会透传到目标服务器
