# IpBlack API 文档

## 接口概述

IpBlack 控制器负责IP黑名单管理，支持对恶意IP的封禁和管理。提供**分级封禁**、**自动解封**机制，支持手动添加和QPS超限自动封禁两种封禁方式。

**接口类型**：基础接口

---

## 封禁等级规范

系统采用 4 级封禁制，等级越高封禁时间越长，自动封禁会根据违规次数逐级升级。

| 等级 | level 值 | 封禁时长 | duration | 触发条件 |
|------|----------|----------|----------|----------|
| 1级 | 1 | 1 小时 | 1 | 首次触发 QPS 警告上限 |
| 2级 | 2 | 24 小时 | 24 | 二次触发 QPS 警告上限 |
| 3级 | 3 | 7 天 | 168 | 三次触发 QPS 警告上限 |
| 永久 | - | 永久 | - | 第四次及以上触发 / 手动设置 |

**自动升级逻辑**：同一IP每次触发自动封禁时，封禁等级在上一级基础上 +1，达到 4 级后保持永久封禁。

## 数据字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 记录ID（主键） |
| ip | string | 被封禁的IP地址（自动转大写） |
| level | int | 封禁等级（1-4） |
| duration | int64 | 封禁时长（小时），永久封禁时为 0 |
| expire_time | int64 | 解封时间戳（Unix秒），永久封禁时为 0 |
| is_permanent | bool | 是否永久封禁 |
| violation_count | int | 累计违规次数 |
| agent | string | 浏览器User-Agent |
| cause | string | 封禁原因 |
| remark | string | 备注说明 |
| create_time | int64 | 创建时间戳 |
| update_time | int64 | 更新时间戳 |
| delete_time | int64 | 软删除时间戳 |

| 状态码 | 使用场景 | 开发注意事项 |

### 9. 添加记录 [基础接口-添加数据]

**请求方式**：POST  
**请求路径**：`/api/ip-black/create`

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| ip | string | 是 | - | 要封禁的IP地址 |
| level | int | 否 | 1 | 封禁等级（1-4），不传则默认 1 级 |
| duration | int64 | 否 | 根据 level 自动计算 | 自定义封禁时长（小时），优先级高于 level |
| expire_time | int64 | 否 | 当前时间 + duration | 自定义解封时间戳（Unix秒） |
| is_permanent | bool | 否 | false | 是否永久封禁，设为 true 时忽略 level 和 duration |
| cause | string | 否 | - | 封禁原因 |
| remark | string | 否 | - | 备注说明 |

**使用示例**：

```json
// 示例1：默认 1 级封禁（1 小时）
{ "ip": "192.168.1.100", "cause": "违规访问" }

// 示例2：3 级封禁（7 天）
{ "ip": "192.168.1.100", "level": 3 }

// 示例3：永久封禁
{ "ip": "192.168.1.100", "is_permanent": true, "cause": "恶意攻击" }

// 示例4：自定义 48 小时封禁
{ "ip": "192.168.1.100", "duration": 48, "remark": "临时封禁" }
```

**响应示例**：

```json
{
  "code": 200,
  "msg": "创建成功！",
  "data": {
    "id": 1
  }
}
```

## 特殊说明

### IP格式处理

- IP地址自动转换为大写格式存储
- 支持IPv4地址格式

### 缓存机制

- 查询接口支持缓存
- 数据更新后自动清除相关缓存
- 缓存键：`[GET][ip-black][column]`

### 自动解封机制

- 系统后台协程每 **5 分钟** 自动检查所有封禁记录
- 当 `expire_time < 当前时间` 且 `is_permanent = false` 时自动软删除
- 永久封禁记录不会被自动解封，需手动删除

### 自动封禁机制

当 IP 达到 QPS 警告上限（在 `config` 表中通过 `SYSTEM_QPS_BLOCK` 配置）时，系统会自动将该 IP 加入黑名单：

- 首次触发：1 级封禁（1 小时）
- 二次触发：自动升级为 2 级（24 小时）
- 三次触发：自动升级为 3 级（7 天）
- 四次及以上：自动升级为永久封禁
- 每次封禁时 `violation_count`（违规次数）自动 +1

### 与 IP 白名单的关系

- 加入 `IpWhite` 白名单的 IP 不受 QPS 限制和自动封禁影响
- 白名单 IP 即使被手动加入黑名单，也不会被中间件拦截
- 推荐将可信的内部服务、测试IP加入白名单
