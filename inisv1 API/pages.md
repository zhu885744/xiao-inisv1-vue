## 页面管理 API

### 接口概述

页面管理控制器，提供独立页面的 CRUD 操作及数据统计功能。

### 状态码规范

| 状态码 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 成功响应数据 |
| 201 | 创建成功 | 资源创建成功 |
| 202 | 接受请求 | 无实际功能接口 |
| 204 | 无数据 | 查询无结果或无可操作数据 |
| 400 | 请求参数错误 | 参数校验失败、数据操作失败 |
| 401 | 未授权 | 未登录或 token 无效 |
| 405 | 方法不允许 | 调用了不存在的方法 |
| 500 | 服务器错误 | 服务器内部错误 |

### 接口列表

#### 1. 获取指定页面 [one]

**接口类型**: 获取指定

**请求方式**: `GET`

**请求地址**: `/api/pages/one`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 否 | 页面ID |
| key | string | 否 | 页面标识 |
| field | string | 否 | 返回字段 |

**响应示例**:

```json
{
  "code": 200,
  "msg": "数据请求成功！",
  "data": {
    "id": 1,
    "key": "about",
    "title": "关于我们",
    "content": "<p>关于我们...</p>",
    "views": 1234,
    "create_time": 1699920000
  }
}
```

### 特殊说明

1. **浏览量统计**: 访问页面时自动更新浏览量，使用 IP+UserAgent 去重，每日同一设备只计数一次
2. **经验值奖励**: 登录用户访问页面可获得经验值奖励
3. **发布时间**: 创建时可指定发布时间，默认为当前时间
4. **审核机制**: 根据系统配置决定是否需要审核
