## OAuth 第三方登录 API

### 接口概述

OAuth 第三方登录控制器，支持 QQ 和 GitHub 登录。

### 状态码规范

| 状态码 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 登录成功 |
| 400 | 请求参数错误 | 参数校验失败 |
| 405 | 方法不允许 | 调用了不存在的方法 |
| 500 | 服务器错误 | 第三方接口调用失败 |

### 接口列表

#### 1. QQ 登录 [qq]

**请求方式**: `GET` / `POST`

**请求地址**: `/api/oauth/qq`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| code | string | 是 | OAuth 授权码 |
| state | string | 是 | 状态参数 |

**响应示例**:

```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "nickname": "用户昵称",
    "figureurl_qq_1": "头像URL",
    "openid": "用户唯一标识"
  }
}
```

---

#### 2. GitHub 登录 [github]

**请求方式**: `GET` / `POST`

**请求地址**: `/api/oauth/github`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| code | string | 是 | OAuth 授权码 |

**响应示例**:

```json
{
  "code": 200,
  "msg": "登录成功！",
  "data": {
    "login": "用户名",
    "id": 123456,
    "avatar_url": "头像URL",
    "html_url": "个人主页"
  }
}
```

### 特殊说明

1. **登录流程**: 用户通过前端跳转第三方授权页面，获取 code 后调用此接口换取用户信息
2. **错误处理**: 获取 token 或用户信息失败时返回 500 错误
3. **返回数据**: 直接返回第三方平台返回的用户信息，前端可根据此信息完成注册/登录
