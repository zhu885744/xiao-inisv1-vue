# IpWhite API 文档

## 接口概述

IpWhite（IP白名单）用于管理可信IP。加入白名单的 IP 将豁免于以下限制：

- **QPS 限流**：白名单 IP 不受请求频率限制
- **自动封禁**：白名单 IP 不会因 QPS 警告被自动加入黑名单
- **手动黑名单**：白名单 IP 即使被手动加入黑名单，也不会被中间件拦截

**典型用途**：
- 内部服务IP（如爬虫、定时任务、API调用方）
- 测试环境IP
- 开发人员办公网络IP
- 合作方的固定出口IP

## 状态码规范

| 状态码 | 使用场景 | 开发注意事项 |
|--------|----------|--------------|
| 200 | 接口调用成功且数据返回正常 | 所有正常响应必须返回200 |
| 204 | 接口调用成功但无数据返回 | 空数据场景必须返回204 |
| 400 | 请求参数错误或业务校验失败 | 参数校验失败 |
| 405 | 请求方法不允许 | 调用了未定义的method |
| 500 | 服务器异常 | 生产环境屏蔽具体错误 |

## 数据字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 记录ID（主键） |
| ip | string | 白名单IP地址 |
| remark | string | 备注说明（如"内网服务"、"测试环境"等） |
| create_time | int64 | 创建时间戳 |
| update_time | int64 | 更新时间戳 |
| delete_time | int64 | 软删除时间戳 |

**请求方式**：GET  
**请求路径**：`/api/ip-white/one`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | int | 是 | 记录ID |

## 特殊说明

### 白名单与黑名单的优先级

- **白名单优先级更高**：如果一个IP同时在白名单和黑名单中，白名单优先，该IP不会被拦截。
- **建议场景**：内部服务、可信合作方、固定办公网络等高频访问IP，建议加入白名单以免误封。

### 白名单生效范围

加入白名单后，该IP将获得以下豁免：

1. **QPS 限流豁免**：不受单接口和全局限速影响
2. **QPS 自动封禁豁免**：不会因频繁请求触发自动封禁
3. **黑名单拦截豁免**：即使在黑名单中也不会被拦截

### 缓存机制

- 白名单查询支持缓存
- 缓存键：`[GET][ip-white][column]`
- 数据更新后自动清除相关缓存
- 缓存状态由 `CacheToml` 配置控制

### 安全建议

- **谨慎添加**：白名单中的IP拥有较高访问权限，添加前请确认IP归属和用途
- **详细备注**：remark字段应详细说明用途和责任人，便于后期维护
- **定期审查**：建议定期审查白名单列表，移除不再需要的IP
- **避免使用CIDR**：当前仅支持单一IP，如需放行网段请在网络层（防火墙）配置
