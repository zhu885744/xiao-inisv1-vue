## 友情链接分组管理 API

### 接口概述

友情链接分组管理控制器，提供链接分组的 CRUD 操作及数据统计功能。

### 状态码规范

| 状态码 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 成功响应数据 |
| 201 | 创建成功 | 资源创建成功 |
| 204 | 无数据 | 查询无结果或无可操作数据 |
| 400 | 请求参数错误 | 参数校验失败、数据操作失败 |
| 401 | 未授权 | 未登录或 token 无效 |
| 403 | 禁止访问 | 无权限操作 |
| 405 | 方法不允许 | 调用了不存在的方法 |
| 500 | 服务器错误 | 服务器内部错误 |

### 接口列表

#### 1. 获取指定分组 [one]

**接口类型**: 获取指定

**请求方式**: `GET`

**请求地址**: `/api/links-group/one`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 否 | 分组ID |
| field | string | 否 | 返回字段，逗号分隔 |
| onlyTrashed | bool | 否 | 只查询已删除数据 |
| withTrashed | bool | 否 | 包含已删除数据 |

**响应示例**:

```json
{
  "code": 200,
  "msg": "数据请求成功！",
  "data": {
    "id": 1,
    "name": "推荐链接",
    "description": "优质友情链接",
    "avatar": "",
    "create_time": 1699920000
  }
}
```

### 特殊说明

1. **缓存机制**: 所有 GET 请求支持缓存，数据修改后自动清除相关缓存
2. **字段处理**: map 和 slice 类型字段会自动序列化为 JSON 字符串存储
3. **权限控制**: 普通用户只能查看审核通过的链接，管理员可查看全部
