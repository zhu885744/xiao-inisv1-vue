# Banner 接口文档

## 接口概述

`banner` 控制器用于管理网站轮播图/横幅广告，支持横幅的创建、查询、更新和删除操作。包含XSS防护和权限控制。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | 支持15个基础接口：one、all、rand、count、sum、min、max、column、remove、delete、clear、restore、save、create、update |
| **特殊接口** | 无 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **201** | 创建成功 | 新增数据成功 |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败、包含恶意代码 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

## 接口列表

### 1. GET 请求接口

#### 1.1 获取单个横幅 [基础接口-获取指定]

- **路径**: `/api/banner/one`
- **方法**: `GET`
- **描述**: 根据条件获取单个横幅

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 横幅ID |
| `field` | string | 否 | 返回字段，逗号分隔 |
| `where` | json | 否 | 条件查询 |
| `like` | json | 否 | 模糊查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "id": 1,
        "title": "横幅标题",
        "url": "https://example.com",
        "image": "/uploads/banner.jpg",
        "create_time": 1699900000
    }
}
```

### 2. POST 请求接口

#### 2.1 保存横幅 [基础接口-保存数据]

- **路径**: `/api/banner/save`
- **方法**: `POST`
- **描述**: 保存横幅（新增或更新）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 横幅ID，为空时新增 |
| `title` | string | **是** | 横幅标题 |
| `content` | string | 否 | 横幅内容描述 |
| `url` | string | 否 | 跳转链接 |
| `image` | string | **是** | 图片路径 |
| `target` | string | 否 | 打开方式（_blank/_self） |
| `start_time` | int | 否 | 开始时间戳 |
| `end_time` | int | 否 | 结束时间戳 |
| `remark` | string | 否 | 备注 |
| `json` | json | 否 | JSON数据 |
| `text` | string | 否 | 文本内容 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "创建成功！",
    "data": {
        "id": 1
    }
}
```

**失败响应** (400):
```json
{
    "code": 400,
    "msg": "内容包含恶意代码，禁止提交！",
    "data": null
}
```

## 特殊说明

### 1. XSS防护
- 创建和更新时会检测标题、内容、URL、图片、备注、文本字段是否包含XSS攻击
- 包含恶意代码的内容会被拒绝提交
- 字符串字段会自动进行HTML清理

### 2. 缓存策略
- 所有查询接口均支持缓存
- 数据修改后会自动清除相关缓存

### 3. 权限控制
- 普通用户只能操作自己创建的横幅
- 管理员可以操作所有横幅
