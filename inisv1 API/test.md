# Test 测试控制器接口文档

## 接口概述

`test` 控制器用于提供各种测试功能的 API 接口，包括请求测试、文件上传测试、支付回调测试等。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **GET 接口** | request - 请求测试 |
| **POST 接口** | return-url、notify-url、request、upload |
| **PUT 接口** | request - 请求测试 |
| **DELETE 接口** | request - 请求测试 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

---

## 接口列表

### 1. GET 请求接口

#### 1.1 请求测试

- **路径**: `/api/test/request`
- **方法**: `GET`
- **描述**: 测试 GET 请求，返回请求信息

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 测试ID |
| `where` | string | 否 | 查询条件 |
| `or` | string | 否 | 或条件 |
| `like` | string | 否 | 模糊查询 |
| `cache` | string | 否 | 是否使用缓存 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "method": "GET",
        "params": {},
        "headers": {}
    }
}
```

## 特殊说明

### 1. 测试用途
- 该控制器主要用于开发和测试阶段
- 生产环境建议禁用或限制访问

### 2. 文件上传
- 支持标准文件上传
- 返回文件后缀和上传结果信息

### 3. 支付回调
- `return-url` 和 `notify-url` 用于支付集成测试
- 实际使用时应替换为真实业务逻辑
