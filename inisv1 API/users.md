# Users 用户管理接口文档

## 接口概述

`users` 控制器用于管理系统用户数据，支持用户的 CRUD 操作、状态管理、邮箱/手机号修改及账户注销功能。所有接口均支持缓存优化和权限控制。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | one、all、rand、count、sum、min、max、column、remove、delete、clear、restore、save、create、update |
| **特殊接口** | email（修改邮箱）、phone（修改手机号）、status（修改状态）、destroy（注销账户） |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **201** | 创建成功 | 新增数据成功、验证码发送成功 |
| **202** | 接受请求 | 请求已接受（无实际功能） |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

## 接口列表

### 1. GET 请求接口

#### 1.1 获取单个用户 [基础接口-获取指定]

- **路径**: `/api/users/one`
- **方法**: `GET`
- **描述**: 根据条件获取单个用户

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 用户ID |
| `email` | string | 否 | 邮箱 |
| `field` | string | 否 | 返回字段，逗号分隔 |
| `where` | json | 否 | 条件查询 |
| `like` | json | 否 | 模糊查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "id": 1,
        "nickname": "张三",
        "email": "zhangsan@example.com",
        "avatar": "",
        "create_time": 1699900000

}
```

**权限说明**: 非管理员无法查看其他用户的 account、email、phone 字段

## 特殊说明

### 1. 缓存策略
- 所有查询接口均支持缓存
- 数据修改后会自动清除相关缓存

### 2. 权限控制
- 密码字段始终不返回
- 非管理员无法查看其他用户的敏感字段（account、email、phone）
- 普通用户只能修改自己的信息

### 3. 数据验证
- 创建和更新时会进行参数验证
- 字符串字段会进行 XSS 检测和 HTML 清理
- 密码会自动加密存储

### 4. 安全限制
- 禁止删除或禁用系统管理员账户（ID=1）
- 用户不能删除自己
- 验证码有频率限制（60秒间隔，每日10次）

### 5. 账户注销
- 注销前会自动清理用户关联数据
- 注销后用户数据被永久删除，不可恢复
