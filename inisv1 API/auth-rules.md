# Auth Rules 接口文档

## 接口概述

`auth-rules` 控制器用于管理权限规则（API接口权限），支持规则的创建、查询、更新和删除操作。规则包含路由、方法、类型等信息，并自动生成路由哈希。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | 支持15个基础接口：one、all、rand、count、sum、min、max、column、remove、delete、clear、restore、save、create、update |
| **特殊接口** | 无 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **201** | 创建成功 | 新增数据成功 |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

## 接口列表

### 1. GET 请求接口

#### 1.1 获取单个规则 [基础接口-获取指定]

- **路径**: `/api/auth-rules/one`
- **方法**: `GET`
- **描述**: 根据条件获取单个权限规则

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 规则ID |
| `field` | string | 否 | 返回字段，逗号分隔 |
| `where` | json | 否 | 条件查询 |
| `like` | json | 否 | 模糊查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "id": 1,
        "name": "获取文章列表",
        "route": "/api/article/all",
        "method": "GET",
        "type": "api",
        "hash": 123456789,
        "create_time": 1699900000
    }
}
```

### 2. POST 请求接口

| 参数名 | 类型 | 必填 | 说明 |

| `method` | string | **是** | HTTP方法（GET/POST/PUT/DELETE） |

| `json` | json | 否 | JSON数据 |
| `text` | string | 否 | 文本内容 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "创建成功！",
    "data": {
        "id": 1
    }
}
```

## 特殊说明

### 1. 哈希自动生成
- 创建和更新规则时，会自动计算 `hash` 字段
- 哈希格式: `[METHOD]route` 的32位哈希值
- `method` 会自动转换为大写

### 2. 字段处理
- `json` 和数组类型字段会自动序列化

### 3. 缓存策略
- 所有查询接口均支持缓存
- 数据修改后会自动清除相关缓存
