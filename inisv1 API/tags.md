# Tags 标签管理接口文档

## 接口概述

`tags` 控制器用于管理系统标签数据，支持标签的 CRUD 操作及数据统计功能。所有接口均支持缓存优化。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | 支持13个基础接口：one、all、rand、count、sum、min、max、column、remove、delete、clear、restore、save、create、update |
| **特殊接口** | 无 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **201** | 创建成功 | 新增数据成功 |
| **202** | 接受请求 | 请求已接受（无实际功能） |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

## 接口列表

### 1. GET 请求接口

#### 1.1 获取单个标签 [基础接口-获取指定]

- **路径**: `/api/tags/one`
- **方法**: `GET`
- **描述**: 根据条件获取单个标签

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 标签ID |
| `field` | string | 否 | 返回字段，逗号分隔 |
| `where` | json | 否 | 条件查询 |
| `like` | json | 否 | 模糊查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "id": 1,
        "name": "技术",
        "description": "技术相关标签",
        "avatar": "",
        "create_time": 1699900000
    }
}
```

## 特殊说明

### 1. 缓存策略
- 所有查询接口均支持缓存
- 数据修改后会自动清除相关缓存
- 缓存名称包含请求参数的哈希值

### 2. 字段处理
- `json` 和数组类型字段会自动序列化
- 字符串字段会进行 XSS 检测和 HTML 清理

### 3. 数据验证
- 创建和更新时会进行参数验证
- 支持自定义验证规则
