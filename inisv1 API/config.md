# Config 接口文档

## 接口概述

`config` 控制器用于管理系统配置项，支持配置的创建、查询、更新和删除操作。包含系统配置自动检测功能（如API_KEY自动创建密钥）。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | 支持15个基础接口：one、all、count、column、remove、delete、clear、restore、save、create、update（缺少：sum、min、max、rand） |
| **特殊接口** | 无 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **201** | 创建成功 | 新增数据成功 |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

## 接口列表

### 1. GET 请求接口

#### 1.1 获取单个配置 [基础接口-获取指定]

- **路径**: `/api/config/one`
- **方法**: `GET`
- **描述**: 根据key获取单个配置项

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `key` | string | **是** | 配置键名 |
| `field` | string | 否 | 返回字段，逗号分隔 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "key": "ARTICLE",
        "value": "1",
        "json": {...},
        "create_time": 1699900000
    }
}
```

**权限说明**: 普通用户无法查看 SYSTEM_ 前缀的配置

#### 1.2 获取所有配置 [基础接口-获取全部]

- **路径**: `/api/config/all`
- **方法**: `GET`
- **描述**: 分页获取配置列表

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `page` | int | 否 | 页码，默认1 |
| `limit` | int | 否 | 每页数量 |
| `order` | string | 否 | 排序字段，默认 `create_time desc` |
| `field` | string | 否 | 返回字段，逗号分隔 |
| `where` | json | 否 | 条件查询 |
| `like` | json | 否 | 模糊查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "data": [...],
        "count": 10,
        "page": 1
    }
}
```

**权限说明**: 普通用户无法查看 SYSTEM_ 前缀的配置

#### 1.3 查询数量 [基础接口-查询数量]

- **路径**: `/api/config/count`
- **方法**: `GET`
- **描述**: 查询配置数量

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `where` | json | 否 | 条件查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "查询成功！",
    "data": 10
}
```

#### 1.4 查询列 [基础接口-查询列]

- **路径**: `/api/config/column`
- **方法**: `GET`
- **描述**: 获取指定字段列表

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `field` | string | **是** | 字段名 |
| `keys` | string | 否 | 指定key列表，逗号分隔 |
| `order` | string | 否 | 排序字段 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": ["ARTICLE", "COMMENT"]
```

## 特殊说明

### 1. 系统配置保护
- 普通用户无法查看 SYSTEM_ 前缀的配置项
- 只有管理员可以访问系统级配置

### 2. 配置监听
- 更新配置后会触发 `watch()` 函数
- 当开启 SYSTEM_API_KEY 且无可用密钥时，会自动创建一个密钥

### 3. 缓存策略
- 所有查询接口均支持缓存
- 数据修改后会自动清除相关缓存

### 4. 配置标识
- 使用 key 作为唯一标识而非 id
- 删除和恢复操作使用 keys 参数而非 ids
