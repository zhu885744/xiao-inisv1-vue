## 友情链接管理 API

### 接口概述

友情链接管理控制器，提供友情链接的 CRUD 操作、状态检测等功能。

### 状态码规范

| 状态码 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 成功响应数据 |
| 201 | 创建成功 | 资源创建成功 |
| 204 | 无数据 | 查询无结果或无可操作数据 |
| 400 | 请求参数错误 | 参数校验失败、数据操作失败 |
| 401 | 未授权 | 未登录或 token 无效 |
| 403 | 禁止访问 | 无权限操作 |
| 405 | 方法不允许 | 调用了不存在的方法 |
| 500 | 服务器错误 | 服务器内部错误 |

### 接口列表

#### 1. 获取指定链接 [one]

**接口类型**: 获取指定

**请求方式**: `GET`

**请求地址**: `/api/links/one`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 否 | 链接ID |
| field | string | 否 | 返回字段，逗号分隔 |
| status | bool | 否 | 是否检测链接状态 |

**响应示例**:

```json
{
  "code": 200,
  "msg": "数据请求成功！",
  "data": {
    "id": 1,
    "nickname": "示例网站",
    "url": "https://example.com",
    "description": "示例描述",
    "avatar": "",
    "group": 1,
    "online": true,
    "responseTime": 120
  }
}
```

### 特殊说明

1. **链接状态检测**: 通过 `status` 参数可检测链接可用性，返回 `online` 和 `responseTime` 字段
2. **XSS防护**: 创建/更新链接时自动检测并清理恶意代码
3. **审核机制**: 普通用户创建的链接需要审核，管理员可直接发布
4. **权限控制**: 用户只能管理自己创建的链接，管理员可管理所有链接
