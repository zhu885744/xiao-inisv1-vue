# Article 接口文档

## 接口概述

`article` 控制器用于管理文章内容，支持文章的创建、编辑、查询、删除等操作。包含浏览量统计、经验值奖励机制，支持审核流程、权限控制和**草稿功能**。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | 支持15个基础接口：one、all、rand、count、sum、min、max、column、remove、delete、clear、restore、save、create、update |
| **特殊接口** | 浏览量更新、经验值奖励（自动触发） |

### 状态字段说明

| 状态值 | 说明 |
| :--- | :--- |
| **0** | 草稿 - 文章未发布，不对外展示 |
| **1** | 发布 - 文章已发布，可以正常展示 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **201** | 创建成功 | 新增数据成功 |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

## 接口列表

### 1. GET 请求接口

#### 1.1 获取单个文章 [基础接口-获取指定]

- **路径**: `/api/article/one`
- **方法**: `GET`
- **描述**: 根据条件获取单篇文章

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 文章ID |
| `field` | string | 否 | 返回字段，逗号分隔 |
| `where` | json | 否 | 条件查询 |
| `like` | json | 否 | 模糊查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "id": 1,
        "title": "文章标题",
        "abstract": "文章摘要",
        "content": "文章内容",
        "views": 100,
        "create_time": 1699900000
    }
}
```

**特殊说明**: 获取文章时会自动更新浏览量和用户经验值

#### 1.2 获取所有文章 [基础接口-获取全部]

### 2. POST 请求接口

#### 2.1 保存文章 [基础接口-保存数据]

- **方法**: `POST`
- **描述**: 保存文章（新增或更新），支持保存为草稿

| 参数名 | 类型 | 必填 | 说明 |

| `json` | json | 否 | JSON数据 |
| `text` | string | 否 | 文本内容 |
| `publish_time` | int | 否 | 发布时间戳 |
| `status` | int | 否 | 状态：0-草稿，1-发布（默认1） |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "创建成功！",
    "data": {
```

## 特殊说明

### 1. 浏览量统计
- 浏览量采用 IP+User-Agent 去重机制
- 同一设备一天内访问同一篇文章只计一次浏览

### 2. 经验值奖励
- 用户查看文章时会自动获得经验值
- 经验值记录类型为 `visit`，绑定类型为 `article`

### 3. 审核流程
- 根据系统配置决定是否开启审核
- 未开启审核时，文章创建后直接通过
- 管理员可以设置 `top` 和 `audit` 字段

### 4. 缓存策略
- 所有查询接口均支持缓存
- 数据修改后会自动清除相关缓存

### 5. 字段处理
- `covers` 和 `tags` 支持数组格式，会自动序列化
- `json` 类型字段会自动序列化
