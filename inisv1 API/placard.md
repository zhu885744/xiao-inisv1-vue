## 公告管理 API

### 接口概述

公告管理控制器，提供公告的 CRUD 操作及数据统计功能。

### 状态码规范

| 状态码 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 成功响应数据 |
| 201 | 创建成功 | 资源创建成功 |
| 202 | 接受请求 | 无实际功能接口 |
| 204 | 无数据 | 查询无结果或无可操作数据 |
| 400 | 请求参数错误 | 参数校验失败、数据操作失败 |
| 405 | 方法不允许 | 调用了不存在的方法 |
| 500 | 服务器错误 | 服务器内部错误 |

### 接口列表

#### 1. 获取指定公告 [one]

**接口类型**: 获取指定

**请求方式**: `GET`

**请求地址**: `/api/placard/one`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| id | int | 否 | 公告ID |
| field | string | 否 | 返回字段 |

**响应示例**:

```json
{
  "code": 200,
  "msg": "数据请求成功！",
  "data": {
    "id": 1,
    "title": "系统升级通知",
    "content": "<p>系统将于今晚升级...</p>",
    "type": "notice",
    "url": "",
    "create_time": 1699920000
  }
}
```

### 特殊说明

1. **XSS防护**: 创建/更新公告时自动检测并清理恶意代码
2. **公告类型**: 可通过 `type` 字段区分不同类型的公告（如通知、警告等）
3. **跳转链接**: 支持配置跳转链接及打开方式（_blank/_self）
