# Comment 接口文档

## 接口概述

`comment` 控制器用于管理评论系统，支持评论的创建、查询、更新和删除操作。包含评论速率限制、敏感词过滤、中文验证、邮件通知等功能。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | 支持15个基础接口：one、all、rand、count、sum、min、max、column、remove、delete、clear、restore、save、create、update |
| **特殊接口** | flat - 获取评论及回复树结构 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **201** | 创建成功 | 新增数据成功 |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败、内容包含恶意代码 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **429** | 请求频繁 | 评论过于频繁 |
| **500** | 服务器错误 | 系统内部错误 |

### 1. GET 请求接口

#### 1.1 获取单个评论 [基础接口-获取指定]

- **路径**: `/api/comment/one`
- **方法**: `GET`
- **描述**: 根据条件获取单个评论

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 评论ID |
| `pid` | int | 否 | 父评论ID |
| `bind_id` | int | 否 | 绑定资源ID |
| `bind_type` | string | 否 | 绑定类型（article/page） |
| `field` | string | 否 | 返回字段，逗号分隔 |
| `where` | json | 否 | 条件查询 |
| `like` | json | 否 | 模糊查询 |
| `withTrashed` | bool | 否 | 是否包含已删除数据 |
| `onlyTrashed` | bool | 否 | 是否只查询已删除数据 |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "数据请求成功！",
    "data": {
        "id": 1,
        "pid": 0,
        "content": "评论内容",
        "bind_id": 1,
        "bind_type": "article",
        "uid": 1,
        "create_time": 1699900000
    }
}
```

## 特殊说明

### 1. 评论配置检查
- 创建评论前会检查文章/页面的评论配置
- 如果评论功能被禁用，会拒绝创建

### 2. 速率限制
- 支持配置评论速率限制（时间窗口内最大评论数）
- 超过限制会返回429状态码

### 3. 内容验证
- 支持配置评论内容最大长度限制
- 支持配置必须包含中文验证
- 支持敏感词过滤

### 4. XSS防护
- 评论内容会检测XSS攻击
- 包含恶意代码的内容会被拒绝提交
- 内容会自动进行HTML清理

### 5. 经验值奖励
- 用户发表评论会获得经验值奖励

### 6. 邮件通知
- 支持配置邮件通知功能
- 评论会通知文章/页面作者
- 回复评论会通知被回复的用户

### 7. 评论树结构
- `flat` 接口返回嵌套的评论树结构
- 支持递归获取所有层级的回复
