# Upgrade 升级控制器接口文档

## 接口概述

`upgrade` 控制器用于系统升级管理，支持主题升级和系统升级功能，同时提供版本管理的 CRUD 操作。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **GET 接口** | list、detail - 获取版本列表和详情 |
| **POST 接口** | theme、system、create、update、delete - 升级操作和版本管理 |
| **PUT 接口** | 暂不支持 |
| **DELETE 接口** | 暂不支持 |

---

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **202** | 接受请求 | 请求已接受（无实际功能） |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **404** | 未找到 | 版本不存在 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

## 接口列表

### 1. GET 请求接口

#### 1.1 获取版本列表

- **路径**: `/api/upgrade/list`
- **方法**: `GET`
- **描述**: 获取版本列表（需要登录）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `type` | string | 否 | 类型筛选：app、theme |
| `status` | int | 否 | 状态筛选 |
| `version` | string | 否 | 版本号搜索 |
| `page` | int | 否 | 页码，默认1 |
| `limit` | int | 否 | 每页数量，默认20 |
| `sort` | string | 否 | 排序字段，默认 create_time |
| `order` | string | 否 | 排序方式：asc、desc，默认 desc |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "获取版本列表成功！",
    "data": {
        "list": [...],
        "total": 100,
        "page": 1,
        "limit": 20
    }
}
```

#### 1.2 获取版本详情

- **路径**: `/api/upgrade/detail`
- **方法**: `GET`
- **描述**: 获取版本详情（需要登录）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | **是** | 版本ID |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "获取版本详情成功！",
    "data": {
        "id": 1,
        "version": "1.0.0",
        "type": "app",
        "content": "更新说明",
        "url": "http://example.com/update.zip",
        "status": 1,
        "create_time": 1699900000
    }
}
```

### 2. POST 请求接口

#### 2.1 主题升级

- **路径**: `/api/upgrade/theme`

- **描述**: 升级主题（需要登录）

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |

{

    "msg": "升级成功！",
    "data": {

        "version": "1.0.0",

## 特殊说明

### 1. 升级流程
- **主题升级**: 下载压缩包 → 删除旧文件 → 解压新文件 → 删除压缩包
- **系统升级**: 下载压缩包 → 解压覆盖 → 删除压缩包 → 重启进程

### 2. 版本类型
- **app**: 系统版本
- **theme**: 主题版本

### 3. 安全性
- 所有接口需要登录验证
- 系统升级会自动重启进程，需谨慎操作
- 升级操作建议在维护时段进行

### 4. 远程服务器
- 版本下载地址优先从本地数据库获取
- 如果本地没有，则从远程服务器获取

### 5. 签名验证
- 请求远程服务器时会携带签名信息
- 包含用户IP、User-Agent等信息用于安全验证
