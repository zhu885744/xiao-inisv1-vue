## QPS告警管理 API

### 接口概述

QPS告警管理控制器，提供QPS告警记录的 CRUD 操作及数据统计功能。

**核心功能**：当系统检测到IP频繁请求超过阈值时，自动记录告警信息；当同一IP在设定时间窗口内累计告警次数达到封禁阈值时，**自动将该IP加入黑名单**。

---

### 状态码规范

| 状态码 | 含义 | 使用场景 |
| :--- | :--- | :--- |
| 200 | 请求成功 | 成功响应数据 |
| 201 | 创建成功 | 资源创建成功 |
| 202 | 接受请求 | 无实际功能接口 |
| 204 | 无数据 | 查询无结果或无可操作数据 |
| 400 | 请求参数错误 | 参数校验失败、数据操作失败 |
| 405 | 方法不允许 | 调用了不存在的方法 |
| 500 | 服务器错误 | 服务器内部错误 |

### 特殊说明

#### 1. QPS告警机制

QPS警告由两个配置项控制，均在 `config` 表中通过 key 存储：

| 配置Key | 类型 | 说明 |
| :--- | :--- | :--- |
| `SYSTEM_QPS` | 基础配置 | 控制是否启用QPS限流，以及单接口/全局请求速度阈值 |
| `SYSTEM_QPS_BLOCK` | 封禁配置 | 控制时间窗口内触发多少次警告后自动封禁IP |

**SYSTEM_QPS 配置项**（value 为 true/false，json 中存储具体参数）：

```json
{
  "point": 10,   // 单接口限速：每秒最多请求数
  "global": 50   // 全局限速：每秒最多请求数
}
```

**SYSTEM_QPS_BLOCK 配置项**（value 为 true/false，json 中存储具体参数）：

```json
{
  "count": 10,      // 在时间窗口内累计警告次数阈值，达到后自动封禁
  "second": 600     // 时间窗口（秒），即最近 N 秒内的警告次数
}
```

**工作流程**：

```
用户请求过于频繁（超过QPS阈值）
    ↓
系统返回 429 错误（请求过于频繁）
    ↓
同时向 qps_warn 表写入一条警告记录（记录IP、路径、User-Agent）
    ↓
检查该IP在最近 N 秒内（second 配置）的警告总数
    ↓
警告数 >= count 配置 → 自动将该IP加入 ip_black 黑名单
```

#### 2. IP封禁等级升级逻辑

被自动封禁的IP会根据历史违规次数自动升级封禁等级：

| 违规次数 | 封禁等级 | 封禁时长 |
| :--- | :--- | :--- |
| 第 1 次 | 1 级 | 1 小时 |
| 第 2 次 | 2 级 | 24 小时 |
| 第 3 次 | 3 级 | 7 天 |
| 第 4 次及以上 | 永久 | 永久封禁 |

每次自动封禁时 `violation_count` 字段自动 +1。

#### 3. 与 IP 白名单的关系

- 加入 `IpWhite` 白名单的 IP **不受 QPS 限制和自动封禁影响**
- 白名单IP即使触发了QPS警告，也不会被自动加入黑名单
- 推荐将可信的内部服务、测试IP加入白名单

#### 4. 自动解封

- 系统后台协程每 **5 分钟** 自动检查所有黑名单记录
- 到期的临时封禁自动软删除（从黑名单中移除）
- 永久封禁需手动删除才能解封

#### 5. 缓存机制

- 查询接口支持缓存
- 缓存键：`[GET][qps-warn][column]`
- 数据更新后自动清除相关缓存
